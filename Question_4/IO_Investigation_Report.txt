5. File Descriptor Observations
Command Used:
ls -l /proc/779/fd

Output:

lrwx------ 1 sudarshan_patil_h_j sudarshan_patil_h_j 64 Jul  3 15:57 1 -> /dev/pts/0
lrwx------ 1 sudarshan_patil_h_j sudarshan_patil_h_j 64 Jul  3 15:57 2 -> /dev/pts/0
lr-x------ 1 sudarshan_patil_h_j sudarshan_patil_h_j 64 Jul  3 15:57 3 -> '/mnt/c/Users/Darshan Patil H J/Desktop/Linux-Assignment/Question_4/app.log'
lr-x------ 1 sudarshan_patil_h_j sudarshan_patil_h_j 64 Jul  3 15:57 4 -> anon_inode:inotify

Observation:
The /proc/<PID>/fd directory was used to inspect the file descriptors of the tail process.
Descriptor 0 represents standard input, descriptor 1 represents standard output, and descriptor 2 represents standard error, all attached to the terminal device /dev/pts/0.
Descriptor 3 points to app.log, which confirms that the file is currently open by the process.
Descriptor 4 points to an inotify object, which is used by tail -f to monitor changes to the file.
This demonstrates that Linux represents open files and I/O resources through numbered file descriptors.


6. Output Redirection Result
Commands Used:
echo "This line is written using output redirection." > output.txt
cat output.txt

Output:
This line is written using output redirection.

Observation:
The > operator was used to redirect the standard output of the echo command into a file named output.txt.
This created the file and stored the text inside it instead of printing it to the terminal.
The cat command was then used to verify the contents of the file.
This demonstrates how Linux allows command output to be redirected from standard output to a regular file.


7. Append Redirection Result
Commands Used:
echo "This line is appended to the same file." >> output.txt
cat output.txt

Output:
This line is written using output redirection.
This line is appended to the same file.

Observation:
The >> operator was used to append additional text to the existing output.txt file.
Unlike >, which overwrites the file, >> preserves the existing contents and adds new data at the end.
The cat command confirmed that both lines were present in the file after the append operation.
This demonstrates how Linux supports appending output to an existing file without losing previously stored data.


8. Error Redirection Result
Commands Used:
cat missing_file.txt 2> error.txt
cat error.txt

Output:
cat: missing_file.txt: No such file or directory

Observation:
The command attempted to read a file that does not exist, which generated an error message.
The 2> operator redirected the standard error stream of the cat command into the file error.txt.
The contents of error.txt were then displayed to verify that the error message had been captured successfully.
This demonstrates how Linux separates standard output and standard error into different streams and allows them to be redirected independently.

10. Explanation of How Linux Manages File I/O
Linux manages file I/O using file descriptors, which are small integer values associated with open files and I/O streams.
Every process typically starts with three standard file descriptors: 0 for standard input, 1 for standard output, and 2 for standard error.
When a process opens a file, Linux assigns it another file descriptor and maintains the relationship through the kernel.
Commands like lsof and the /proc/<PID>/fd interface help inspect these open file descriptors.
Linux also supports redirection operators such as >, >>, and 2> to route output and errors into files instead of the terminal.
This design makes file handling flexible and efficient for both system processes and user applications.
