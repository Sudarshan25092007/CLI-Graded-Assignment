1. Umask Value
Command Used:
umask

Output:
0022

Observation:
The umask command was used to check the default permission mask of the current Linux session.
A umask value of 0022 means that newly created directories normally receive 755 permissions and newly created files receive 644 permissions in a standard Linux filesystem.
This command is useful because it explains the default access rights applied during file and directory creation.
The umask value was recorded before setting up the project workspace.

7. Explanation of How Permissions Protect Project Data
Explanation:
Linux permissions are used to control which users can read, write, or execute files and directories.
For a secure project workspace, directories are commonly assigned permission 750 so that the owner has full access, the group has read and execute access, and others have no access.
Files are commonly assigned permission 640 so that the owner can read and modify them, the group can only read them, and others cannot access them at all.
This type of permission setup helps protect project documents, source files, and reports from unauthorized viewing or modification by other users on the system.

Conclusion:
A project workspace was successfully created with separate folders for documentation, source files, and reports.
The workspace owner and file metadata were verified using ls and stat commands, and the default umask value was recorded.
Although chmod commands were used to apply secure permissions, the displayed permissions remained unchanged because the workspace was created on a Windows-mounted filesystem in WSL.
Conceptually, the intended secure permission model for this workspace is 750 for directories and 640 for files, which would provide better protection for project data on a standard Linux filesystem.
